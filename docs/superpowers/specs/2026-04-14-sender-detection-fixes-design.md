# Sender Detection Fixes — Design Spec
*2026-04-14*

## Overview

5 точечных фиксов в `src/services/email-analyzer.js` для улучшения извлечения ФИО, должности
и фильтрации ложных артикулов. Выявлено на реальных письмах от 11.04.2026.

**Scope:** только баг-фиксы в существующем файле. Рефакторинг на отдельные модули — отдельная задача.

---

## Fix 1 — ФИО: формат "Имя Фамилия И." + компания как контекст

**Симптом:** письмо от artur@oilgis.ru → ФИО "Не определено"  
**Подпись:** `--\nАлик Шарифгалиев М.\nООО ОйлГИС\n8 903 351 9285`  
**Место:** `extractFullNameFromBody`, ~строка 2350

### Проблемы

1. `cyrillic2words` требует только строчных букв после первой → не матчит "Алик Шарифгалиев М." (заглавная "М.")
2. `hasContext` проверяет только телефон/email/ключевые слова должности → "ООО ОйлГИС" не является контекстом

### Правки

**a) Добавить паттерн `cyrillicWithInitial`:**
```js
// "Имя Фамилия И." или "Фамилия И.О." — два слова + один инициал с точкой
const cyrillicWithInitial = /^([А-ЯЁ][а-яё]{1,19})\s+([А-ЯЁ][а-яё]{1,19})\s+([А-ЯЁ]\.(?:\s*[А-ЯЁ]\.)?)$/.test(line);
```

Добавить в условие: `if (!cyrillic2words && !cyrillicWithInitial && !latin2words && !surnameInitials) continue;`

**b) Расширить `hasContext` — добавить юридические формы как контекст:**
```js
const hasContext = /(?:\+7|8[-\s(]|tel:|mob:|e-?mail:|@|менеджер|инженер|директор|специалист|начальник|заместитель|руководитель|manager|engineer|sales|ООО|АО|ОАО|ЗАО|ПАО|ИП\b|ГК\b|НПО|НПП|Филиал|ФГУП|МУП)/i.test(neighbor);
```

## Fix 2 — ФИО: org-unit display name не используется как имя человека

**Симптом:** письмо от `филиал "НТИИМ" <SFKZC@ntiim.ru>` → ФИО "филиал НТИИМ" вместо "Бастрыкова Мария"  
**Место:** `extractSender`, ~строка 1191

### Проблема

`fromName` = "филиал «НТИИМ»" передаётся напрямую как `fullName`. Это название подразделения,
а не человека. Реальное имя "Бастрыкова Мария" есть в подписи тела письма.

### Правки

**a) Добавить `isOrgUnitName(str)`:**
```js
const ORG_UNIT_PREFIXES = /^(?:филиал|отдел|цех|управление|департамент|служба|лаборатория|сектор|группа|подразделение|division|department|branch)/i;
// Также: display name из ОДНОГО слова полностью в верхнем регистре (аббревиатура типа "СФКЗЦ")
function isOrgUnitName(str) {
  if (!str) return false;
  if (ORG_UNIT_PREFIXES.test(str.trim())) return true;
  // Одно слово в CAPS или с «» скобками — скорее всего орг. единица, не человек
  if (/^[«"]?[А-ЯЁA-Z0-9\-«»"']+[»"]?$/.test(str.trim()) && !/\s/.test(str.trim())) return true;
  return false;
}
```

**b) Изменить логику fullName в `extractSender`:**
```js
// До:
const fullName = fromName || extractFullNameFromBody(body) || inferNameFromEmail(fromEmail) || "Не определено";

// После:
const nameFromDisplay = (!isOrgUnitName(fromName)) ? fromName : null;
const fullName = nameFromDisplay || extractFullNameFromBody(body) || inferNameFromEmail(fromEmail) || "Не определено";
```

**c) Расширить паттерн "С уважением" — позиция между приветствием и именем:**

```js
// До — только один optional блок (компания):
/(?:С уважением|Благодарю|Спасибо)[,.\s]*\n?\s*(?:(?:ООО|АО|...)\s+...)?([А-ЯЁ][а-яё]+\s+...)/i

// После — добавить optional строку должности перед именем:
const signatureWithPosition = body.match(
  /(?:С уважением|Благодарю|Спасибо)[,.\s]*\r?\n\s*(?:[^\n]{3,60}\r?\n\s*)?([А-ЯЁ][а-яё]+(?:\s+[А-ЯЁ][а-яё]+){1,2})/i
);
if (signatureWithPosition) {
  const candidate = signatureWithPosition[1].trim();
  // Убедиться что это не компания (нет ООО/АО и т.п.)
  if (!/^(?:ООО|АО|ОАО|ЗАО|ПАО|ИП|ГК|НПО|ФГУП)/i.test(candidate)) {
    return candidate;
  }
}
```

## Fix 3 — ФИО: форма не перезаписывает полное display name

**Симптом:** письмо от `Алиса Гурьева <gurevaa18@mail.ru>`, цитата содержит "Имя посетителя: Алиса" → ФИО "Алиса"  
**Место:** `analyzeEmail`, ~строка 388

### Проблема

Цитированная форма robot@ перезаписывает `fromName` = "Алиса Гурьева" на "Алиса" (одно слово из формы).

### Правка

```js
// До:
if (quotedRobotFormData.name) fromName = quotedRobotFormData.name;

// После:
if (quotedRobotFormData.name) {
  const currentWords = fromName.trim().split(/\s+/).filter(Boolean).length;
  const formWords = quotedRobotFormData.name.trim().split(/\s+/).filter(Boolean).length;
  // Перезаписываем только если форма даёт больше информации
  if (formWords > currentWords) fromName = quotedRobotFormData.name;
}
```

То же самое применить к `robotFormData.name` (прямая форма, ~строка 372) — там тоже может быть регрессия.

## Fix 4 — ЭДО-идентификатор не попадает в артикулы

**Симптом:** "2BM-0278106553-2012052808163395382630000000000" → попадает в Артикулы как "2BM-0278106553-201205"  
**Место:** `isGarbage`, ~строка 3792

### Проблема

Существующий паттерн `/^[02]?[A-ZА-Я]{1,2}-\d{10}(?:-\d{9})?$/i` покрывает только короткие IDs с ровно 1-2 сегментами.
Полный Diadoc ID может быть `2BM-INN(10 цифр)-TIMESTAMP(20+ цифр)`.

### Правка

```js
// Расширить до: 2BM-/BM- с любым количеством цифровых сегментов
// Также добавить контекстный паттерн: строки рядом с "ЭДО"/"Диадок"/"идентификатор"
if (/^[02]?[A-ZА-ЯЁ]{1,3}-\d{7,}(?:-\d+)*$/i.test(normalized)) return true;
```

Дополнительно добавить в `ARTICLE_GARBAGE_PATTERNS` (статический массив ~строка 275):
```js
// Diadoc/СБИС идентификаторы: 2BM-..., BM-..., любой оператор
/^[02]?[A-ZА-ЯЁ]{1,3}-\d{7,}(?:-\d+)*$/i,
```

## Fix 5 — Должность: fallback-паттерн для позиций из подписи

**Симптом:** "начальник СФКЗЦ" не извлекается в поле Должность  
**Место:** `extractPosition`, ~строка 2391

### Проблема

`detectionKb.matchFieldBest("position", body)` полагается на KB. Специфические должности 
вида "начальник [аббревиатура]" могут не быть в KB.

### Правка

Добавить regex-fallback после KB:
```js
function extractPosition(body) {
  const position = detectionKb.matchFieldBest("position", body);
  if (position) return cleanup(position);

  // Fallback: "Должность: X" явный лейбл
  const labelMatch = body.match(/(?:должность|position|звание)\s*[:\-–]\s*([^\n,]{3,80})/i);
  if (labelMatch) return cleanup(labelMatch[1]);

  // Fallback: строка подписи вида "начальник|заместитель|зав. + [отдел/аббревиатура]"
  const signaturePositionMatch = body.match(
    /(?:^|\n)\s*((?:начальник|заместитель\s+начальника?|главный\s+(?:инженер|технолог|бухгалтер|специалист)|зав\.|заведующий|руководитель\s+(?:отдела|направления|группы)|ведущий\s+(?:инженер|специалист)|генеральный\s+директор|коммерческий\s+директор|технический\s+директор)\s*[^\n,]{0,50})/im
  );
  if (signaturePositionMatch) return cleanup(signaturePositionMatch[1]);

  return null;
}
```

## Новые паттерны ФИО (дополнительные улучшения)

В рамках тех же правок добавить в `extractFullNameFromBody` новые паттерны:

1. **"Фамилия Имя" в Genitive/обратном порядке** — "Бастрыкова Мария" (фамилия первой):
   Уже покрывается `cyrillic2words`, но нужно убедиться что `surnameFirst` ("Бастрыкова Мария") проходит. ✓ Работает через structured block — проблема была только в `hasContext`.

2. **Инициалы перед фамилией** "М. Шарифгалиев" — добавить паттерн:
   ```js
   const initialsBeforeSurname = /^([А-ЯЁ]\.\s*){1,2}[А-ЯЁ][а-яё]{2,}$/.test(line);
   ```

3. **Latin имена в Cyrillic-письмах** — уже есть `latinSignatureMatch`, но добавить вариант без приветствия (просто "John Smith" в блоке подписи) аналогично `cyrillic2words`.

## Тестовые случаи

| Входные данные | Ожидаемый результат |
|---|---|
| `fromName="", body="--\nАлик Шарифгалиев М.\nООО ОйлГИС"` | ФИО: "Алик Шарифгалиев М." |
| `fromName="филиал НТИИМ", body="С уважением,\nначальник СФКЗЦ\nБастрыкова Мария\n+7..."` | ФИО: "Бастрыкова Мария", Должность: "начальник СФКЗЦ" |
| `fromName="Алиса Гурьева", quotedFormName="Алиса"` | ФИО: "Алиса Гурьева" (форма не перебивает) |
| `article="2BM-0278106553-2012052808163395382630000000000"` | Не в артикулах |
| `body="начальник СФКЗЦ\nБастрыкова Мария"` | Должность: "начальник СФКЗЦ" |
