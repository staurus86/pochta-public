## Decision 1 — Сворачиваемый сайдбар (иконки)

**Поведение:**
- По умолчанию: развёрнут (240px)
- Кнопка `‹` сворачивает до 48px (только иконки)
- В свёрнутом состоянии: логотип → только иконка `P`, текст пунктов → скрыт, бейджи → скрыты
- Tooltip с названием пункта при hover на иконку (title-атрибут)
- Кнопка меняет направление: `‹` ↔ `›`
- Состояние сохраняется в `localStorage` (`pochta_sidebar_collapsed`)

**Responsive breakpoints:**
- `≥1280px` — по умолчанию развёрнут
- `960–1279px` — по умолчанию свёрнут до иконок (автоматически)
- `<768px` — сайдбар полностью скрыт, появляется кнопка `☰` в шапке, сайдбар открывается как overlay с backdrop

**CSS-изменения:**
```css
.sidebar { width: 240px; transition: width 0.2s ease; }
.sidebar.collapsed { width: 48px; overflow: hidden; }
.sidebar.collapsed .nav-text,
.sidebar.collapsed .nav-badge,
.sidebar.collapsed .nav-section-label,
.sidebar.collapsed .sidebar-brand .version,
.sidebar.collapsed .logo-text,
.sidebar.collapsed .sidebar-stats,
.sidebar.collapsed .sidebar-footer span { display: none; }
```

**JS:** toggle-кнопка в `.sidebar-brand`, обработчик записывает класс + localStorage.

## Decision 2 — Правая панель (аккордеон)

**Вместо:** плоского списка технических полей (`detail-field` key/value).

**Структура аккордеона:**

```
┌─ Статус-блок (всегда открыт, не сворачивается) ────────┐
│  [✓ / ⚠ / ✗] Готово к CRM / Модерация / Спам          │
│  Уверенность: 87%                        [Заявка]      │
└─────────────────────────────────────────────────────────┘
▾ 👤 Клиент           (открыт по умолчанию)
  Имя, Должность, Компания, ИНН, Телефон, CRM-инфо

▾ 📦 Заявка           (открыт по умолчанию)
  Бренд, Тип, Позиций, Артикулы (chips)

▸ ⚙ Диагностика      (закрыт по умолчанию)
  Технические поля: confidence, sources, conflicts, issues
```

**Кнопки действий** (всегда видны, под аккордеоном):
- `✓ Передать в CRM` (зелёная, полная ширина)
- `Спам` + `Уточнить` (серые, половина ширины каждая)

**Реализация:** состояние секций хранится в объекте `{client: true, request: true, diagnostics: false}`. Клик по заголовку секции — toggle. Иконка `▾`/`▸` отражает состояние.

**Диагностика (скрытая секция)** содержит прежние технические поля: overallConfidence, primaryIssue, fields с source/confidence, conflicts, issues. Нетехническая аудитория её не видит.

## Decision 3 — Фильтры Входящих: две строки

**Строка 1 — Действия:**
```
[✉ Получить письма] [Удалить все] [CSV] [XLSX] [🔍 Поиск...              ]
```
Bulk-actions (Заявка / Подтвердить / Спам / Поставщик / Удалить / Снять) появляются в этой же строке при выборе чекбоксов — вытесняя кнопки экспорта.

**Строка 2 — Фильтры:**
```
Фильтры: [Все ящики ▾] [Вложения ▾] [Распознание ▾] [LLM ▾]   [Новые ↓] [☐ Треды] [Авто: выкл]
```

**Строка 2 скрывается** на мобиле (<768px) за кнопкой `⚙ Фильтры`.

## Адаптив — полная таблица

| Breakpoint | Сайдбар | Inbox layout | Фильтры строка 2 |
|---|---|---|---|
| `≥1280px` | развёрнут 240px | 3 колонки (260 / 1fr / 300) | видна |
| `960–1279px` | иконки 48px | 3 колонки (260 / 1fr / 280) | видна |
| `768–959px` | иконки 48px | 2 колонки (240 / 1fr), правая панель — overlay по кнопке | кнопка «Фильтры» |
| `<768px` | overlay drawer | 1 колонка: список → клик → full email + панель | кнопка «Фильтры» |

**На мобиле (<768px):**
- Сайдбар: кнопка `☰` в `main-header`, overlay с `backdrop` (полупрозрачный фон)
- Inbox: только список писем; клик открывает письмо full-screen с кнопкой `←` назад
- Правая панель (детали): кнопка `📋 Детали` под телом письма открывает панель аккордеона full-screen с кнопкой `←` назад
- Прогресс-бар: `left: 0` (без смещения на сайдбар)

## Дополнительные улучшения

### Шапка (`main-header`)
- Кнопки `Переанализировать` и `LLM-анализ` — убрать из шапки на страницах, где они не нужны (Дашборд). Оставить только на страницах Входящих / Проектов.
- На мобиле: скрыть лишние кнопки, оставить только заголовок + `project-select`.

### Прогресс-бар
- `left` должен меняться вместе с шириной сайдбара: CSS-переменная `--sidebar-width` обновляется при toggle.

### Детальная панель — ширина
- Desktop: 300px (было 360px) — освобождает место для тела письма
- При свёрнутом сайдбаре: +48px к телу письма
